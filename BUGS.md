# Баг-репорты

## Баг №1. Фильтр по цене применяет некорректные границы

**Краткое описание:**  
При вводе значений "от 10000" и "до 50000" в фильтр цены, в URL подставляются параметры minPrice=15000 и maxPrice=55000. В результате присутствуют объявления с ценой выше 50000.

**Шаги воспроизведения:**
1. Открыть главную страницу https://cerulean-praline-8e5aa6.netlify.app/
2. В поле "От" ввести 10000
3. В поле "До" ввести 50000
4. Дождаться автоматического обновления списка

**Фактический результат:**
- URL содержит параметры `?minPrice=15000&maxPrice=55000`
- В выдаче есть объявление с ценой 54974 (выше ожидаемых 50000)

**Ожидаемый результат:**
- URL должен содержать `minPrice=10000&maxPrice=50000`
- Все отображаемые объявления должны иметь цену в диапазоне 10000–50000

**Серьёзность:** P2 (средняя) — функциональность фильтрации работает с искажением пользовательского ввода.

**Окружение:**
- Браузер: Chromium (Playwright) / Chrome последней версии
- ОС: Windows 11
- Дата: 08.04.2026
- Стенд: https://cerulean-praline-8e5aa6.netlify.app/

## Баг №6. Фильтр по цене изменяет нулевые значения на другие числа

**Краткое описание:**  
При вводе в поля «От» и «До» значения `0` фильтр не применяет диапазон `minPrice=0&maxPrice=0`, а подставляет другие числовые границы (например, `10000` и `6000`).

**Шаги воспроизведения:**
1. Открыть главную страницу https://cerulean-praline-8e5aa6.netlify.app/
2. В поле "От" ввести `0`
3. В поле "До" ввести `0`
4. Дождаться автоматического применения фильтра

**Фактический результат:**  
- URL изменяется, но содержит параметры `minPrice=10000&maxPrice=6000` (или другие значения, отличные от `0`).
- Выдаются объявления с ценами в подставленном диапазоне.

**Ожидаемый результат:**  
- Если ввод нуля разрешён, фильтр должен примениться с `minPrice=0&maxPrice=0`, показав пустой список или сообщение "Объявления не найдены".
- Если ввод нуля запрещён, поля должны очищаться или не позволять ввод, а URL не должен изменяться.

**Серьёзность:** P3 (низкая) – нулевой диапазон редко используется, но некорректная обработка вводит пользователя в заблуждение.
